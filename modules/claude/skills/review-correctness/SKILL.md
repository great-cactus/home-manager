対象の科学技術文章（論文・レポート・技術文書）の「正確性」を専門的に校閲し、修正提案をファイルに保存せよ。

## 良い科学技術文章の基準

良い科学技術文章は以下を満たす：
- **シンプルで明解な論理**（大枠→詳細、つながりが自明、前提共有）
- **解釈ブレがない**（同一対象に同一用語、文法ミスなし）

## スコープ

このスキルは「正しい/誤り」が客観的に判断できる問題のみを担当する。
解釈や好みの問題（曖昧さ、スタイル、構成）は review-clarity / review-coherence の担当。

## 1. 深層分析

校閲を開始する前に、文章全体を通読して深く考えること。思考過程を `<thinking>` ブロックに明示せよ。

## 2. チェック項目

### 文法エラー ★重要★
- 主語と動詞の一致
- 時制の一貫性（過去形 vs 現在形）
- 冠詞（a, an, the）の適切な使用
- 前置詞の正確さ
- 複数形・単数形の使い分け

#### 学術・技術文書での適切な時制

| セクション | 推奨時制 |
|-----------|---------|
| Abstract | 過去形（研究内容）、現在形（結論） |
| Introduction | 現在形（一般的事実）、過去形（先行研究） |
| Methods | 過去形 |
| Results | 過去形 |
| Discussion | 現在形（解釈）、過去形（結果への言及） |

#### よくある文法ミス

| 避けるべき表現 | 推奨表現 |
|--------------|---------|
| is existed | exists |
| can be able to | can / is able to |
| more higher | higher |
| the value is increased | the value increases |

### 数式の正確性
- 次元解析（単位の整合性）
- 数学的な導出の正しさ
- 記号の定義と一貫した使用
- 方程式番号の参照の正確さ

### 数値・単位
- 有効数字の適切さ
- SI単位の正しい表記
- 数値の整合性（本文・図・表間）
- 誤差・不確かさの表記

### 引用・参照の正確性
- 参照番号が正しいか
- 式・図・表への参照が存在するか
- 参照先の内容と本文の記述が一致するか

### 図表の技術的正確性
- 軸ラベルと単位が正しいか
- キャプションが正確か
- 本文との数値整合性

## 3. 提出前の自己検証

以下をすべて確認してから保存すること：
- [ ] 対象文書を全て読んだ
- [ ] 全チェック項目を確認した
- [ ] 「十分な問題を見つけた」で止めず網羅的に探した
- [ ] 各問題に明確な場所と修正案がある

禁止：
1. 自己検証チェックリスト未完了で保存
2. 「問題なさそう」でセクションをスキップ
3. 場所と修正案のない曖昧な問題を報告
4. 独自の判断で確認できない事実を断定する（不確かな場合は「?」付きでフラグ）

必須：
1. 問題がないセクションも明示的に「問題なし」と記載
2. 不確かな場合は「?」付きでフラグ

## 4. 修正提案の保存

すべての修正提案を `.claude/tmp/review-correctness.md` に書き込め。ファイルが存在しない場合は新規作成せよ。

各修正案は以下のフォーマットで記述すること：

```markdown
## [重要度] 問題の種類 - 場所

#### 要修正箇所
（英語で記述：該当箇所の原文）

#### 修正が必要な理由
（日本語で記述）

#### 修正提案
（英語で記述）

#### 対応
- [ ] 承認
- [ ] 拒否
- [ ] 条件付承認

##### modification
（承認後、ここに最終的な修正内容を記載）

---
```

重要度の分類：
- **Critical**: 技術的に誤りで、結論や再現性に影響する
- **Major**: 明確な文法エラー、数値・単位の不整合
- **Minor**: 軽微な表記ミス、スタイルの修正

## 5. 完了報告

保存完了後、以下の形式でユーザーに報告せよ：

```
[Correctness] 校閲完了
Critical: X件 / Major: Y件 / Minor: Z件
保存先: .claude/tmp/review-correctness.md
```
