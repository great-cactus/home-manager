# Concise Writing（必要十分への圧縮）

文章を，伝えたい内容を保ったまま最短の表現に仕上げるスキル。対象は校閲依頼されたテキストと，Claude自身が生成する説明・文書の両方。

## 中核原理：不変性テスト

まず「伝えたい命題の集合 M」を確定する。そのうえで文章の各要素（語・句・文・段落）に次の3操作を仮想的に施す：

1. **削除**しても M が変わらないか
2. **別の表現に言い換え**ても M が変わらないか（＝その表現自体に意味がない）
3. **順序を入れ替え**ても M が変わらないか（＝その位置・接続に論理的役割がない）

いずれの操作でも M が不変なら，その要素は情報を担っていない。**削る**。

これは Strunk & White の "Omit needless words"（every word must tell）の操作的な定義であり，目標状態は**必要十分**：M を伝えるのに必要な要素はすべて残り（必要），M に寄与しない要素は1つもない（十分性の裏返し）。

## 手順：トップダウンに4階層で削る

大きい単位から削るほうが効率がよい（段落ごと削れば語句の推敲が不要になる）。

### Step 0: M の確定

削る前に，この文章が伝えるべき命題を列挙する。長い文書なら storyline-outliner スキルでアウトライン化すると M が明確になる。**Step 0 を飛ばして語句いじりから始めない**——何が必要かを決めずに十分性は判定できない。

### Step 1: 文書レベル——読者が必要としない情報を削る

- 読者の目的・前提知識に照らして不要な情報（既知の事項，目的に無関係な背景・脱線・余談）を削る
- 同じ内容が複数箇所に現れていれば1箇所に統合する（重複＝MECE違反）
- 図表に示した情報を本文で繰り返さない

### Step 2: 段落レベル——段落削除テスト

各段落に問う：「この段落を丸ごと削除してはならない理由を一言で言えるか」。言えなければ削る。段落内では，トピックセンテンスを支えない文を削るか移動する。

### Step 3: 文レベル——前置きの切除と一文一義

- **前置き（slow windup）の切除**：「In this study, we...」「〜について述べると」「ここで注意すべきは〜ということである」など，本題の前のファンファーレは削って本題から始める
- 一文一義に分割すると，各文に不変性テストをかけやすくなる
- 文単位の削除テスト：その文を削って M が欠けるか

### Step 4: 語句レベル——Williams の6原則＋言い換え圧縮

Joseph Williams（*Style: Lessons in Clarity and Grace*）の6原則：

1. ほとんど意味を持たない語を削る
2. 他の語と意味が重複する語を削る
3. 他の語から含意される語を削る
4. 句を1語に置き換える
5. 否定形を肯定形に変える（not different → similar）
6. 役に立たない形容詞・副詞を削る

加えて Lanham の Paramedic Method 由来の言い換え圧縮（削除でなく短い等価表現への置換）：

- 名詞化の解消：「分析を行う」→「分析する」，"conduct an examination" → "examine"
- 受動態→能動態（行為者が自明または既出のとき）
- "there is/are ... that" 構文の再構成："There is evidence that supports..." → "Evidence supports..."

#### 日本語の冗長表現（掃引リスト）

| 冗長 | 圧縮 |
|---|---|
| 〜するということ | 〜すること／削除 |
| 〜することができる | 〜できる |
| 〜を行う／〜を実施する | 動詞化（測定を行う→測定する） |
| 〜という形で／〜のような形 | 削除または「〜で」 |
| まず最初に | まず |
| 基本的に，ある意味，いわゆる | 削除（限定の意図がなければ） |
| 〜であると考えられる | 〜と考えられる／〜である（ヘッジ判断は後述） |

#### 英語の冗長表現（掃引リスト）

| 冗長 | 圧縮 |
|---|---|
| in order to | to |
| due to the fact that | because |
| it should be noted that | （削除） |
| in this study, we | we（多くの場合 in this study ごと削除） |
| is able to | can |
| a number of | several / many |
| at this point in time | now |

### Step 5: 検収

自己チェックリスト（後述）を通し，必要なら **Lard Factor**（圧縮率 = 削った語数／元の語数）を報告する。技術文書では 30–50% の圧縮が珍しくない。

## 削ってはならないもの（必要性の境界）

不変性テストの「M が変わらない」判定を厳密に行うこと。次は**削ると M が変わる**ので残す：

- **意味を担う限定子**：approximately, may, under X conditions,「〜の場合に限り」。削ると主張が強くなり，科学的には誤りになる。空虚な強意語（very, 非常に）とは区別する
- **読者に必要な定義・前提**：変数定義（math-notation スキルの要求）や，初学者向けの用語説明・前提（記述順序スキルが補うもの）は M の伝達に必要な要素であり，冗長ではない
- **トーン・敬意の表現**：メールの丁寧表現は意味を担う（email-english-proofreading スキルの判断が優先）
- **意図的な強調の反復**：結論部での要点再掲など，機能が明確な反復

## 自己チェックリスト

1. **文削除テスト**：どの1文を削っても M のいずれかが欠けるか（欠けない文が残っていれば未完）
2. **段落削除テスト**：各段落の存在理由を一言で言えるか
3. **冗長表現掃引**：上のリストの表現が残っていないか
4. **過剰削除チェック**：圧縮後の文章から M がすべて復元できるか。限定子・定義を削っていないか
5. **意味ドリフトチェック**：言い換えた箇所で主張の強さ・範囲が変わっていないか
6. **数式記号チェック**：出力に数式が1本でも含まれる場合，圧縮後の文章に対して math-notation スキルの変数定義監査（全記号に登場の直前後の定義があるか）を再実行する。定義文・定義句（「ここで T は温度」等）は圧縮の対象外

## アンチパターン

### 1. 過剰削除（telegraphic style）

❌ 「触媒は活性を示した」（条件を削った結果，過大主張に）
✅ 「触媒は400 °C以上で活性を示した」

短さは目的ではなく，必要十分が目的。条件・限定子は M の一部。

### 2. 要約との混同

このスキルは情報を**落とさない**。M の一部を捨てて短くするのは要約であり，別タスク。圧縮後の文章から M がすべて読み取れなければ失敗。

### 3. 言い換えによる意味のドリフト

「〜と考えられる」→「〜である」は短くなるが，推定を断定に変えており M が変わる。圧縮は等価変形に限る。

### 4. 箇条書き化による命題の消失

文章を体言止めの箇条書きに変えると短く見えるが，主張（誰が・何を・どうした）が消えて M が劣化する。短縮の手段として安易な箇条書き化を使わない。

## 適用例

❌ 元（54字）：
「まず最初に，本研究において我々が行った実験について述べると，測定を行うことが可能であったということが分かった。」

✅ 圧縮後（11字）：
「実験により測定できた。」

（Lard Factor ≈ 80%。「まず最初に」→冗長，「本研究において〜述べると」→前置き，「行った実験について」→「実験により」，「測定を行うことが可能であった」→「測定できた」，「ということが分かった」→削除）

## 他スキルとの関係

- **記述順序スキル（unfamiliar-topic-explanation）**：順序スキルが構成（何をどの順で言うか）を決め，本スキルが各要素を必要十分に圧縮する。順序スキルが要求する前提・用語説明は「必要」側なので削らない。**説明を生成するときは常に両方を適用する**
- **math-notation**：変数定義の要求は本スキルの削減判断より**常に優先**する。数式を含む下書きでは，圧縮の後（Step 5 のチェック6）に math-notation の定義監査を再実行してから出力する
- **storyline-outliner**：Step 0 の M 抽出に使う
- **scientific-english-proofreading / email-english-proofreading**：それぞれの校閲後，本スキルの観点で最終掃引する。トーン・登録（register）の判断は各スキルが優先
- **Claude自身の出力**：ユーザへの説明・回答・文書を生成するときも，出力前に Step 2–5 を自分の下書きに適用する
