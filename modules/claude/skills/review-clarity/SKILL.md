対象の科学技術文章（論文・レポート・技術文書）の「明瞭さ」を専門的に校閲し、修正提案をファイルに保存せよ。

## 良い科学技術文章の基準

良い科学技術文章は以下を満たす：
- **シンプルで明解な論理**（大枠→詳細、つながりが自明、前提共有）
- **解釈ブレがない**（同一対象に同一用語、文法ミスなし）

## 1. 深層分析

校閲を開始する前に、文章全体を通読して深く考えること。思考過程を `<thinking>` ブロックに明示せよ。

## 2. チェック項目

### 用語の一貫性 ★重要★
- 同じ概念に対して異なる用語が使われていないか
- 用語の定義が明確か
- 略語と正式名称の使い分けが一貫しているか

### 初出時定義の確認 ★重要★
- 技術用語が初めて登場する箇所で適切に説明されているか
- 略語は初出時に正式名称と共に導入されているか

### 代名詞の対応明確化 ★重要★
- **this, it, they, these, those** の指示対象が唯一に定まるか
- 関係代名詞（which, that, who）の先行詞が明確か

### 表記揺れの検出 ★重要★
- 同一概念に複数の表記が混在していないか

### 文間のつながりの自明性 ★重要★
- 文と文の接続が明確か
- 接続詞や遷移文が適切に使われているか
- 読者が「なぜこの文が今出てくるのか」と感じないか

### 一文一メッセージ ★重要★
- 一つの文に複数のメッセージが詰め込まれていないか
- 長い文は分割すべきか

### 暗黙的話題転換の検出 ★重要★
- 文レベルで話題が突然変わっていないか
- 話題転換時に適切な導入があるか

## 3. 提出前の自己検証

以下をすべて確認してから保存すること：
- [ ] 対象文書を全て読んだ
- [ ] 全チェック項目を確認した
- [ ] 「十分な問題を見つけた」で止めず網羅的に探した
- [ ] 各問題に明確な場所と修正案がある

禁止：
1. 自己検証チェックリスト未完了で保存
2. 「問題なさそう」でセクションをスキップ
3. 場所と修正案のない曖昧な問題を報告

必須：
1. 問題がないセクションも明示的に「問題なし」と記載
2. 不確かな場合は「?」付きでフラグ

## 4. 修正提案の保存

すべての修正提案を `.claude/tmp/review-clarity.md` に書き込め。ファイルが存在しない場合は新規作成せよ。

各修正案は以下のフォーマットで記述すること：

```markdown
## [重要度] 問題の種類 - 場所

#### 要修正箇所
（英語で記述：該当箇所の原文）

#### 修正が必要な理由
（日本語で記述）

#### 修正提案
（英語で記述）

#### 対応
- [ ] 承認
- [ ] 拒否
- [ ] 条件付承認

##### modification
（承認後、ここに最終的な修正内容を記載）

---
```

重要度の分類：
- **Critical**: 意味が変わる、または理解を妨げる曖昧さ
- **Major**: 用語の不一致、代名詞の曖昧さ、つながりの不明確さ
- **Minor**: 軽微な表記揺れ、スタイルの改善

## 5. 完了報告

保存完了後、以下の形式でユーザーに報告せよ：

```
[Clarity] 校閲完了
Critical: X件 / Major: Y件 / Minor: Z件
保存先: .claude/tmp/review-clarity.md
```
