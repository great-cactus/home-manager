# 数式記述規則（LaTeX / Typst 共通）

数式を書く・直すときは、以下の規則を **LaTeX でも Typst でも同一の方針** で適用する。
表記の意味論（立体か斜体か等）は処理系に依らず一定であり、実装コマンドだけが処理系ごとに異なる。

判断の根拠は ISO 80000-2（Quantities and units — Part 2: Mathematics）に置く。
迷ったときは「その記号が表しているのは *量/変数* か *固定的な意味を持つもの* か」を問う。前者は斜体、後者は立体。

---

## 1. 立体 / 斜体の使い分け（ISO 80000-2）

原則：**量・変数を表す記号は斜体（italic）、固定的な意味を持つ記号は立体（roman / upright）。**

| 概念 | 字形 | 根拠・例 |
| --- | --- | --- |
| 変数（スカラー量） `x, y, T, p` | 斜体 | 値が文脈で変わる量 |
| 文脈上の定数とみなすパラメータ `a, b` | 斜体 | ISO は「定数とみなせるパラメータも斜体」 |
| 走り番号・添字変数 `i, j, k`（Σ の添字等） | 斜体 | running number は斜体 |
| 一般の関数記号 `f, g` | 斜体 | 文脈依存の関数は斜体 |
| 定義済み関数 `sin, cos, exp, ln, log, lim, det, max, sgn, Re, Im` | **立体** | 固定的意味を持つ関数 |
| 数学定数 `e`（底）, `i`（虚数単位）, 円周率 `π` | 斜体（分野慣習） | ISO は立体だが、流体・燃焼系の慣習に合わせ斜体を既定とする（§6）。立体にするのは下記の演算子のみ |
| **微分演算子 `d`**（`d f / d x`, `∫ … d x`） | **立体** | well-defined operator。Akira の明示要望 |
| 偏微分 `∂`, 変分 `δ`, ナブラ `∇` | **立体** | 演算子 |
| `div, grad, curl, tr` 等の名前付き演算子 | **立体** | 固定的意味 |
| 数字（桁で書いた数）`123`, `0.1` | **立体** | 数字は常に立体 |
| 単位 `K, mm, s, Pa` | **立体**（前に細スペース） | §4 を参照 |

### 添字（subscript / superscript）の使い分け

- **変数・走り番号を表す添字は斜体**：`V_x`, `a_i`, `E^k`。
- **語・略号・固定数を表す添字は立体**：`V_{\mathrm{m}}`（membrane）, `T_{\mathrm{ad}}`（adiabatic）, `c_{\mathrm{p}}`, `Re_{\mathrm{cr}}`, `x_1`(番号は立体)。
- 判断基準：その添字を別の値に置き換えられる（変数）なら斜体、ラベルとして固定（記述語）なら立体。

### ベクトル・行列・テンソル

| 概念 | 字形 | LaTeX | Typst |
| --- | --- | --- | --- |
| ベクトル `a` | 太字斜体（または矢印） | `\bm{a}` / `\vec{a}` | `bold(a)` / `arrow(a)` |
| 行列 `A` | 太字斜体・大文字 | `\bm{A}` | `bold(A)` |
| 2階テンソル `T` | サンセリフ太字斜体 | `\bm{\mathsf{T}}`（isomath `\tensorsym`） | `bold(sans(T))` |

注：`\mathbf` は *立体* 太字で ISO 厳密では非整合（ISO は太字 *斜体*）。ISO 準拠を取るなら `\bm`／`\boldsymbol` を使う。文書全体でどちらかに統一する。

## 2. 分数表記

- **本文中（inline）の数式は スラッシュ形 `A/B`** を使う。行間を乱さないため。
- **独立した一行式（display / ワンライン式）は積み上げ形 `\frac{A}{B}`** を使う。

| | LaTeX | Typst |
| --- | --- | --- |
| inline（スラッシュ形） | `$a/b$`, `$(a+b)/c$` | `$a slash b$` または `$a\/b$`（リテラルの `/`） |
| display（積み上げ形） | `\[ \frac{a}{b} \]` / `\dfrac{a}{b}` | `$ a/b $`（Typst の `/` は自動で積み上がる）／`frac(a, b)` |

注（Typst）：Typst では `/` は inline でも積み上げ分数になる。inline をスラッシュ形に保つには `slash` か `\/` を使い、display では `/` か `frac()` を使う。

- 分子・分母が複数項なら inline でも括弧で曖昧性を消す：`(a + b)/(c + d)`。
- 多段分数・連分数は inline を避け、必ず display にする。

## 3. 変数定義の明示

**登場するすべての変数に定義を与える。** 読者が記号を遡って探さずに済むようにする。

- 原則として、変数の **登場の直前または直後** に定義文を置く。
  - 英語：`…, where T is the temperature and p is the pressure.` / `Let u be the velocity field.`
  - 日本語：`…。ここで T は温度、p は圧力である。`
- **定義を与える式では `:=` を用いて「定義である」ことを明示する。**
  - 例：`Da := τ_flow / τ_chem`（Damköhler 数を定義）／`We define the Damköhler number as Da := τ_flow / τ_chem.`
  - 既知の等式（成り立つ関係）には `=`、新たな定義には `:=` を使い分ける。
- **Nomenclature（記号表）がある文書では、本文中の逐一の定義は不要。** 記号表に載せる。
  - その場合も、記号表との表記ゆれ（同一量に別記号、同一記号に別量）を作らない。

| LaTeX | Typst |
| --- | --- |
| `:=` は `\coloneqq`（`mathtools`/`unicode-math`）。無ければ `\mathrel{:=}` | `:=` または `colon eq`／`eq.def`（≝ を使う場合） |

## 4. 数字・単位の組版（ISO 80000-1 / SI）

- **数字は立体**、**単位記号も立体**。単位の前に **細スペース**（thin space）を入れる：`5 K`, `0.1 mm`, `3.2 × 10^{3} Pa`。
- **小数点は英語文中ではピリオド**（`3.14`）。ISO 本文は小数記号にコンマを使うが、英語の科学出版ではピリオドが標準なので **英語原稿ではピリオドに統一**（コンマ小数は使わない）。
- 桁区切りは細スペースかカンマ（出版先の規定に従う）。同一文書内で混在させない。
- **乗算記号は `×`（U+00D7）**。アルファベットの `x` を乗算に使わない。指数表記は `1.2 × 10^{3}`。

| | LaTeX | Typst |
| --- | --- | --- |
| 細スペース | `\,` | `thin`（または `#h(0.16em)`） |
| 数値＋単位 | `5\,\mathrm{K}` ／ `siunitx`: `\qty{5}{K}` | `5 thin upright("K")` |
| 乗算 | `\times` | `times` |

## 5. 実装早見表（立体化の直接記法）

**カスタムマクロ（`\newcommand`, `\def`, `\DeclareMathOperator` などの独自定義）は使わない。** 投稿先によっては独自マクロを含む原稿を受け付けない（組版・変換工程で展開できない）ため、立体化は **標準コマンドを本文に直接書く**。`\mathrm`, `\operatorname`, `\sin` などの標準コマンドはマクロ定義ではないので問題ない。

**LaTeX**（必要なら `amsmath`, `bm`, `mathtools` を読み込む。いずれも標準パッケージ）

| 概念 | 直接記法 |
| --- | --- |
| 微分の d | `\mathrm{d}`（例：`\int f(x)\,\mathrm{d}x`, `\mathrm{d}y/\mathrm{d}x`） |
| 偏微分・ナブラ | `\partial`, `\nabla`（組み込みで立体） |
| 名前付き演算子 | 組み込み `\sin, \exp, \log, \det, \lim` など。無いものは `\operatorname{div}` を**本文で直書き**（`\DeclareMathOperator` は使わない） |
| 立体の添字 | `_{\mathrm{p}}`, `_{\mathrm{ad}}` |
| ベクトル・行列 | `\bm{a}`, `\bm{A}`（`bm` パッケージ） |
| `:=` | `\coloneqq`（`mathtools`）／無ければ `\mathrel{:=}` |
| `e`, `i`, `π` | 斜体のまま `e`, `i`, `\pi`（何もしない） |

**Typst**

| 概念 | 直接記法 |
| --- | --- |
| 微分の d | `upright("d")`（例：`integral f(x) thin upright("d") x`） |
| 偏微分・ナブラ | `partial`, `nabla`（組み込みで立体） |
| 名前付き演算子 | 組み込み `sin, exp, ln` など。無いものは `op("div")` |
| 立体の添字 | `_"p"`, `_"ad"`（引用符で囲むと立体） |
| ベクトル・行列 | `bold(a)`, `bold(A)` |
| `:=` | `:=` または `eq.def`（≝） |
| `e`, `i`, `π` | 斜体のまま `e`, `i`, `pi`（何もしない） |

Typst は複数文字の識別子を既定で立体、単文字を斜体にする。`#let` による自前ショートカットも、原稿の可搬性を保つため最小限にとどめる。

## 6. 確定した慣習（既定値）

ISO と分野慣習が割れる箇所について、既定を以下に固定する。**文書内では一貫させる**こと。

- **数学定数 `e`・虚数単位 `i`・円周率 `π`：斜体**（流体・燃焼系の慣習）。ISO は立体だが慣習を優先し、`e`, `i`, `\pi` をそのまま斜体で使う。**立体にするのは演算子の `d`（および `∂, δ, ∇`、`div` 等の名前付き演算子）のみ。**
- **ベクトル：太字斜体 `\bm{a}` / `bold(a)`**（矢印 `\vec` は使わない）。
- **大文字ギリシャ**：TeX 既定の立体のまま（ISO 整合で全ギリシャを斜体にするなら `isomath`。既定では不要）。

投稿先が表記を指定する場合（author guidelines）は、その規定を優先し、ここでの既定を上書きする。

## 7. 例

### 例1：微分・分数・定数（立体化）

**修正前（LaTeX）**
```latex
The growth rate is $\frac{dy}{dt} = e^{ikx}$.
```
**修正後**
```latex
The growth rate is $\mathrm{d}y/\mathrm{d}t = e^{ikx}$, where $k$ is the wavenumber.
```
- `\frac{dy}{dt}` は inline なのでスラッシュ形 `\mathrm{d}y/\mathrm{d}t` に
- 立体にするのは微分の `d` のみ（`\mathrm{d}` を直書き。カスタムマクロは使わない）。`e`, `i` は分野慣習で斜体のまま
- `k` の定義を本文に追加（未定義変数を残さない）

### 例2：定義は `:=`、変数の逐次定義

**修正前**
```
We use the Damköhler number Da = tau_flow / tau_chem.
```
**修正後（inline、定義なので `:=`）**
```latex
We define the Damköhler number as $\mathrm{Da} := \tau_{\mathrm{flow}}/\tau_{\mathrm{chem}}$,
where $\tau_{\mathrm{flow}}$ is the characteristic flow time and $\tau_{\mathrm{chem}}$ is the characteristic chemical time.
```
- 定義なので `:=`
- 添字 `flow`, `chem` は記述語なので立体
- 各変数に定義文を付与

### 例3：display 式（積み上げ形）と添字

**Typst**
```typst
$ c_"p" (partial T)/(partial t) = nabla dot (lambda nabla T) + dot(omega)_"T" $
```
- 独立式なので積み上げ形（`/`）
- `∂`, `∇` は立体（Typst 組み込み）
- `c_"p"`（定圧）、`dot(omega)_"T"`（温度のソース項）の添字は記述語なので立体 → Typst では引用符 `_"p"` で囲む
- 直後に `where c_p is the specific heat at constant pressure, λ is the thermal conductivity, …` と定義を続ける

## 8. やってはいけないこと

- 微分の `d` を斜体のまま放置しない（`dy/dx` の `d` は立体）。
- inline 数式で `\frac` を使って行間を膨らませない（inline はスラッシュ形）。
- カスタムマクロ（独自 `\newcommand`/`\DeclareMathOperator` など）に依存しない。投稿先が展開できないことがあるため、`\mathrm{d}`・`\operatorname{div}` 等の標準コマンドを直書きする。
- 定義（新規導入）に `=` を使わない。定義は `:=`。
- 定義のない変数を式に出さない（Nomenclature があれば本文定義は省くが、その場合は記号表に必ず載っていること）。
- 同一量に複数の記号、同一記号に複数の量を割り当てない（表記ゆれ）。
- 立体/斜体・ベクトル記法・小数点記号を文書内で混在させない。一度決めたら通す。
- 投稿規定が本スキルの既定と異なる場合に、規定を無視しない（規定が優先）。
- 頼まれていない式変形・記号変更を勝手にしない。表記の問題は指摘して提案する（Akira の方針）。
