対象の科学技術文章（論文・レポート・技術文書）の「インパクト」を専門的に評価し、修正提案をファイルに保存せよ。

## 良い科学技術文章の基準

良い科学技術文章は以下を満たす：
- **シンプルで明解な論理**（大枠→詳細、つながりが自明、前提共有）
- **解釈ブレがない**（同一対象に同一用語、文法ミスなし）

## スコープ

このスキルは「価値が読者に伝わるか」を担当する：
- インパクトの明瞭さ
- リジェクト要因の検出
- 査読者・読者視点での評価

※正確性、明瞭さ、一貫性はそれぞれ review-correctness / review-clarity / review-coherence の担当

## 1. 深層分析

校閲を開始する前に、文章全体を通読して深く考えること。思考過程を `<thinking>` ブロックに明示せよ。

## 2. チェック項目

### インパクトの明瞭さ ★最重要★
- 研究・技術の貢献が読者に**一読で伝わる**か
- 「何が新しいのか」「なぜ重要か」が明確か
- タイトル・Abstract で価値が伝わるか
- 読者が「だから何？（So what?）」と感じないか

```
NG: 技術的詳細を列挙するだけで、意義が不明瞭
OK: 「シンプルな手法で複雑な問題を解決できた」という価値が明確
```

### 主要メッセージとの整合性 ★重要★
- 主要メッセージが適切に強調されているか
- 文書全体のトーンが主要メッセージと一致しているか

### 読者視点での明瞭さ
- 想定読者にも主要なメッセージが伝わるか
- 前提知識の要求が適切か
- 複雑な概念に十分な説明があるか

### 新規性・貢献の明確さ
- 既存研究・手法との差別化が明確か
- 貢献のインパクトが十分に示されているか
- タイトルとAbstractで新規性が伝わるか

### 先行研究との比較
- 重要な先行研究が引用されているか
- 公平な比較がなされているか
- 最新の関連研究への言及があるか

### 限界・将来展望
- 研究・技術の限界が適切に認識されているか
- 過度な主張をしていないか
- 将来の発展方向が示されているか

## 3. リジェクト要因チェックリスト

### 致命的な問題（即リジェクト相当）
- [ ] **新規性の欠如**: 既存研究・手法と本質的に同じ
- [ ] **方法論の重大な欠陥**: 結論を支持しない
- [ ] **データ・根拠の不足**: 主張を裏付ける証拠が不十分
- [ ] **スコープ外**: 対象読者・ジャーナルの範囲外

### 重大な問題（Major Revision相当）
- [ ] 先行研究との比較不足
- [ ] 仮定の妥当性の説明不足
- [ ] 結論の過度な一般化
- [ ] 方法論の詳細不足
- [ ] **インパクトが不明瞭**

### 軽微な問題（Minor Revision相当）
- [ ] 文献の追加が必要
- [ ] 図表の改善
- [ ] 文章の明瞭化
- [ ] 軽微な技術的修正

## 4. 想定される査読コメントと対策

| 想定コメント | 対策 |
|------------|------|
| "新規性が不明" | Introductionで既存研究との差を明記 |
| "なぜこの手法?" | Methods で手法選択の理由を説明 |
| "データが少ない" | 追加実験または限界として明記 |
| "一般化しすぎ" | 結論を限定的に書き直す |
| "So what?" | インパクト・実用的価値を明示 |

## 5. 提出前の自己検証

以下をすべて確認してから保存すること：
- [ ] 対象文書を全て読んだ
- [ ] 全チェック項目を確認した
- [ ] 「十分な問題を見つけた」で止めず網羅的に探した
- [ ] 各問題に明確な場所と修正案がある

禁止：
1. 自己検証チェックリスト未完了で保存
2. 「問題なさそう」でセクションをスキップ
3. 場所と修正案のない曖昧な問題を報告

必須：
1. 問題がないセクションも明示的に「問題なし」と記載
2. 不確かな場合は「?」付きでフラグ

## 6. 修正提案の保存

すべての修正提案と総合評価を `.claude/tmp/review-impact.md` に書き込め。ファイルが存在しない場合は新規作成せよ。

各修正案は以下のフォーマットで記述すること：

```markdown
## [重要度] 問題の種類 - 場所

#### 要修正箇所
（英語で記述：該当箇所の原文）

#### 修正が必要な理由
（日本語で記述）

#### 修正提案
（英語で記述）

#### 対応
- [ ] 承認
- [ ] 拒否
- [ ] 条件付承認

##### modification
（承認後、ここに最終的な修正内容を記載）

---
```

ファイルの末尾に必ず総合評価を追記すること：

```markdown
## 総合評価

採択可能性: [高/中/低]
インパクトの明瞭さ: [明確/やや不明瞭/不明瞭]

強み:
1. ...
2. ...

要改善点（優先順）:
1. [Critical/Major] ...
2. [Major] ...
3. [Minor] ...

推奨: [Accept as is / Minor revision / Major revision / Reject]
```

重要度の分類：
- **Critical**: インパクトや新規性を根本的に損なう問題
- **Major**: 価値の伝達を妨げる明確な問題
- **Minor**: 改善によりインパクトが高まる提案

## 7. 完了報告

保存完了後、以下の形式でユーザーに報告せよ：

```
[Impact] 校閲完了
Critical: X件 / Major: Y件 / Minor: Z件
採択可能性: [高/中/低]
保存先: .claude/tmp/review-impact.md
```
