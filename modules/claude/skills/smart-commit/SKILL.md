---
name: smart-commit
description: ステージング済みファイルの品質チェック後に自動コミット
---

以下の手順でスマートコミットを実行せよ.

## 1. ステージング確認

`git diff --cached --name-only` でステージ済みファイルを確認する.
ステージ済みファイルがない場合は `git add -A` で全変更をステージングする（.gitignore は尊重される）.

## 2. 品質チェック

ステージ済みファイルの差分（`git diff --cached`）を分析し、以下を検出する:

- 重複した機能（同じロジックが複数箇所に存在）
- 使用されていないコード（未使用の変数・関数・import）
- 可読性の問題（深いネスト、過度に長い関数、不明瞭な命名）

問題が見つかった場合:
- 問題の一覧をユーザーに提示する
- コミットを中止する
- 修正案を提案する

## 3. コミット分割判定

変更内容を分析し、適切な粒度でコミットを分割する:

- `feat`: 新機能
- `fix`: バグ修正
- `refactor`: リファクタリング
- `docs`: ドキュメント
- `chore`: 雑務（設定変更等）
- `perf`: パフォーマンス改善
- `test`: テスト

異なるタイプの変更が混在する場合は、タイプごとに別コミットにする.

## 4. コミット実行

各コミットに対して:
- Conventional Commits 形式（`type: description`）でメッセージを生成する
- メッセージは日本語でも英語でも、変更内容が最も自然に伝わる言語を使う
- `git commit` を実行する
